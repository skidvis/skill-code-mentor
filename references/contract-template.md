## Template Usage Notes

When filling this template:

1. **What You're Building**: Write for a human, not a spec system. Be vivid about what the finished thing is.

2. **Goals**: Mix product goals ("users can log in") with learning goals ("understand how JWTs work") when the project is primarily educational.

3. **Success Criteria**: Write as if you're creating a checklist the developer will literally tick off. Every criterion should be unambiguously pass/fail.

4. **Scope Boundaries**: When in doubt, put it out of scope. Easier to add later than to remove. Be kind about it — explain why it's deferred, not just that it is.

5. **Future Ideas**: This section is a gift. Good ideas shouldn't get lost just because they're out of scope for now.

6. **What You'll Learn**: This is unique to the mentor format. Name the real skills being developed — it sets expectations and helps the developer stay engaged when things get hard.
